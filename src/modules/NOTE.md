# Modules Architecture

## Overview
The `modules/` directory is designed for **feature-based architecture**.

Instead of grouping files by type (pages, services, stores),
we group them by **feature/domain**.

This improves:
- scalability
- maintainability
- team collaboration

---

## Why use modules?

As your project grows, folders like `pages/`, `services/`, and `stores/` become large and hard to manage.

Modules solve this by isolating features.

Example:
- auth
- user
- product
- dashboard
